# 지식 그래프 — 사건 연관 분석기

## 폴더 구조
```
knowledge-graph/
  ├── index.html   ← 브라우저에서 열기
  ├── data.js      ← 코워크(Claude)가 수정하는 데이터
  └── README.md    ← 이 파일
```

## 사용법

### 1. 그래프 열기
`index.html`을 브라우저에서 열면 됩니다.
(더블클릭 or 드래그해서 브라우저에 놓기)

### 2. 사건 분석 요청 (코워크)
코워크에서 Claude에게 이렇게 말하세요:

> "다음 사건을 분석해서 data.js에 추가해줘: [사건 내용]"

Claude가 data.js를 직접 수정합니다.

### 3. 그래프 갱신
브라우저에서 F5 또는 상단 **새로고침** 버튼 클릭.
(파일 변경 감지 시 자동으로 알림 표시)

## data.js 구조

```javascript
window.DATA = {
  "nodes": [
    {
      "id": "ev_고유ID",
      "type": "event",        // event | cause | effect
      "label": "짧은 제목",   // 15자 이내 권장
      "desc": "상세 설명",
      "depth": "단기",        // effect 노드만: 단기|중기|장기
      "x": null,              // null이면 자동 배치
      "y": null
    }
  ],
  "edges": [
    {
      "from": "노드ID",
      "to": "노드ID",
      "shared": true,         // 교차 연결 여부 (선택)
      "label": "연결 설명"    // 교차 연결 설명 (선택)
    }
  ],
  "meta": {
    "created": "2026-04-11",
    "version": "1.0"
  }
}
```

## 노드 ID 규칙
- 핵심 사건: `ev_[주제]_[번호]`  예) `ev_iran_001`
- 원인: `ca_[사건ID]_[번호]`     예) `ca_iran_001`
- 파급효과: `ef_[사건ID]_[번호]` 예) `ef_iran_001`

## 그래프 조작
- **드래그**: 노드 이동
- **스크롤**: 줌 인/아웃
- **클릭**: 노드 상세 정보
- **배경 드래그**: 화면 이동

## 코워크 프롬프트 템플릿

```
이 사건을 분석해서 knowledge-graph/data.js에 추가해줘:

[사건 내용]

추가 규칙:
- 핵심 사건 노드 1개
- 원인 노드 2-3개  
- 파급효과는 단순히 병렬로 나열하지 말고, **1차(단기) → 2차(중기) → 3차(장기/투자수혜)**로 이어지는 **'연쇄적 인과관계(N차 파급효과)'**를 도출해줄 것.
- 연쇄적 파급효과는 일직선뿐만 아니라, **여러 갈래로 뻗어나가는 형태(분기 구조)**도 적극 활용할 것.
- 각 파급효과 노드들이 꼬리를 무는 형태로 Edge(`from` -> `to`)를 명확히 연결할 것.
- 기존 노드와 연관이 있으면 shared 엣지도 추가
- label은 12자 이내
```
